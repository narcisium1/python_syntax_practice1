# Этап 01. Основы tkinter

## Что такое tkinter

`tkinter` - стандартная библиотека Python для создания desktop-приложений с окнами, кнопками, полями ввода и другими графическими элементами.

Главная идея: программа не выполняется сверху вниз до конца, как обычный консольный скрипт. После создания окна запускается главный цикл `mainloop()`. Он ждет действия пользователя: клик, ввод текста, закрытие окна.

## Минимальное окно

Пример из другой области: небольшое приложение для записи книг в личную библиотеку.

```python
import tkinter as tk

root = tk.Tk()
root.title("Домашняя библиотека")
root.geometry("500x300")

label = tk.Label(root, text="Список книг")
label.pack()

root.mainloop()
```

Здесь:

- `Tk()` создает главное окно;
- `title()` задает заголовок окна;
- `geometry()` задает размер;
- `Label` создает текстовую надпись;
- `pack()` размещает виджет внутри окна;
- `mainloop()` запускает ожидание событий.

## Виджеты

Виджет - это элемент интерфейса:

- `Label` - текст;
- `Button` - кнопка;
- `Entry` - поле ввода;
- `Frame` - контейнер для других виджетов;
- `Listbox` - список строк.

Виджет обычно создается в два шага: сначала объект, потом размещение.

```python
button = tk.Button(root, text="Добавить")
button.pack()
```

## Команды кнопок

Кнопка становится полезной, когда ей передают функцию через параметр `command`.
Имя функции передается без скобок, потому что ее нужно вызвать не сразу, а после клика.

```python
def close_app():
    root.destroy()


exit_button = tk.Button(root, text="Закрыть", command=close_app)
exit_button.pack()
```

Метод `destroy()` закрывает окно и завершает главный цикл приложения.

## Почему нельзя смешивать все в одном месте

Даже простое окно быстро становится большим. Если весь код держать в глобальной области, его сложно читать и расширять.

Удобнее создавать класс приложения:

```python
import tkinter as tk


class LibraryApp:
    def __init__(self):
        self.root = tk.Tk()
        self.root.title("Библиотека")

        self.title_label = tk.Label(self.root, text="Мои книги")
        self.title_label.pack()

    def run(self):
        self.root.mainloop()


app = LibraryApp()
app.run()
```

В следующих этапах такой подход поможет отделить создание окна от логики приложения.
